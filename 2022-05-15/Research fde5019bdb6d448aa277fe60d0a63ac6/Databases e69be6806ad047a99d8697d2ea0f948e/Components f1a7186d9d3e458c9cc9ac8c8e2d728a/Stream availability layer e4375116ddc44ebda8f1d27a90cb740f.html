Overview

At the core of the Ceramic protocol sits the stream availability layer. This layer is made up of a network of nodes that are incentivized to make stream data available in the network.

👤Account channel

A user of the network can open up an account channel. This channel keeps track of the latest event all streams of the account an account balance. Nodes in the network use this data structure to keep track of and charge accounts.

🕸️P2P architecture

All nodes in the network choose for themselves which 👤Account channel they subscribe to. In order to find the nodes responsible for a particular account channel, the libp2p DHT is used for routing. Libp2p pubsub is utilized to gossip about new stream 🏔️Tips. Finally an event retrieval market is established which enables nodes to pay each other for the retrieval of events.

⛓️Account sampling sidechain

In order to open an 👤Account channel a user deposits 🔥Fire tokens into the 🥵Kiln smart contract. This credits the users account to be able to emit events to any of the streams it controls. Nodes in the network participate in the consensus of the account sampling sidechain. Instead of every transaction being processed by the chain, each block randomly samples an account channel. The block producer will thus need to retrieve events from this account channel through the 🧮Event retrieval market. Effectively providing an incentive for nodes to store the data of as many account channels as is economically viable for them. In order to participate in the network, nodes, referred to as Ceramic miners stake 🔥Fire tokens into the kiln contract. Miners have two main responsibilities: (1) produce and validate blocks in the sidechain and (2) participating in the event retrieval market, sometimes referred to as event retrieval mining.
