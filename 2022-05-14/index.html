Research

Welcome to our home for technology research at 3Box Labs. This page aims to provide a living overview of Ceramic’s systems design, research goals, and open problems. There is a detailed description of the architecture and its components, current and future implementations of the protocol, and a list of open problems. All research is organized into a four stage process.

Stages

🔸Components

The components DB aspires to be a detailed specification of the entire Ceramic stack. From a top level architecture overview to detailed descriptions of minor components. The information contained within this DB should equip an implementor with everything they need in order to successfully implement one or multiple layers of Ceramic.

🛠️Implementations

The implementations DB is meant as a lightweight overview for all implementations of the three Ceramic layers. Each entry represents a code base and has a reference to each component it implements.

❓Open Problems

The open problems DB aims to highlight all open problems that currently exists within the 🔸Components specs, as well as problems related to specific 🛠️Implementations. Each item in the DB should contain enough information to understand the problem, and may be used as a living document for further research.

💡Proposals

Proposals should contain well specified changes to one or more 🔸Components and/or 🛠️Implementations. For changes that touch changes to components a complete list of changes component by component is required. For changes that touch implementations draft project specs should be included.
