Streams are the core building block of Ceramic. A stream is a linked list of events that can be independently verified by any node. A node in the network that subscribes to a stream validates the event signatures and keep track of the latest 🏔️Tips of the stream.

Event data structure

Genesis event

type GenesisHeader struct {
  controllers [String]
  model optional Bytes
  schema optional String
  family optional String
  tags optional [String]
  unique optional String
}
type GenesisEvent struct {
  header GenesisHeader
}
  • controllers - an array of 🧬DID strings that defines the controllers of the stream (i.e. who is allowed to make updates to it).

Event layout

type Event struct {
  id Link
  prvs optional [ Link ]
  data Any
  prev optional Link
  header optional Any
}
  • id - the CID of the genesis event, or another previous event
  • prvs - the CID(s) of the previous event(s)
  • data - the CID of the event data (or the event data directly for backwards compatibility, CID is preferred because it allows for deletion)
  • prev - the CID of the previous event (legacy support)
  • header - header metadata (legacy support)

Anchor event

type AnchorProof struct {
  chainId String
  txHash Link
  root Link
  blockNumber optional Int
  blockTimestamp optional Int
}

type AnchorEvent struct {
  prvs optional [ Link ]
  proof &AnchorProof
  path String
  id optional Link
  prev optional Link
}
  • chainId - a CAIP-2 string representing a specific blockchain
  • txHash - a CID of a blockchain transaction, e.g. eth-tx
  • root - an IPLD link to a MerkleNode
  • prvs - the CID(s) of the previous event(s)
  • proof - the CID of the AnchorProof
  • path - the path in the merkletree from the root
  • id - the CID of the genesis event, or another previous event
  • prev - the CID of the previous event (legacy support)
